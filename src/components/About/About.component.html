Fixed Broadband Deployment

This application provides a visualization of the residential fixed broadband deployment data collected on FCC Form 477. Facilities-based providers of broadband service report Form 477 data in June and December each year. For more information about Form 477, to download the data, or to see reports generated from the dataset, please refer to the links at right.

Providers report fixed-broadband deployment data at the census block level. Form 477 data are reported using 2010 Census blocks. Providers may not offer service to every home in every block in which they report service. The calculations used to create the graphs treat every location as having service, and may therefore over-estimate broadband coverage, particularly in areas with large census blocks.

The Form 477 data collection relies on 2010 Census blocks as a standard for data collection. Thus all broadband data displayed on this map use 2010 geometries (2010 block, tract and county definitions). Similarly, the definitions of urban and tribal areas rely on the 2010 Census definitions. Search geometries on the Area Summary and Area Comparison pages (state, county, CBSA, census place, Congressional District and tribal areas) are based on updated definitions (see Data Download page for more information).

Block-level estimates for housing units, population and households

The U.S. Census Bureau publishes block-level figures for housing units, population and households only with data from the decennial census. Rather than rely on data from 2010, this site instead relies on block-level estimates that account for increases or decreases in population over time; and the construction of new homes. See Staff Block Estimates for more information.

The methodology used to create those block-level estimates is likely to over-state the number of blocks in which people reside (rather than under-state). As such these estimates may under-estimate population-weighted figures for broadband coverage.

How to Use This Site

This website allows you to explore various facets of fixed-broadband deployment.

The site allows users to choose a from a list of technologies (ADSL, cable modem, FTTP, fixed wireless, satellite and other), and a list of broadband speeds (at least 200 kbps, at least 10 Mbps downstream / 1 Mbps upstream, at least 25/3, at least 100/10 or at least 250/25) in creating the maps and graphs on each page.

  • To see the providers that report service at a given location, go to the Location Summary page and enter and address or click on a location.
  • To see a map of broadband and get summary data about a particular area, go to the Area Summary page, choose the type of area, and use the search bar.
  • To learn more about the technologies and speeds reported by each provider, or to compare providers' footprints, go to the Provider Detail page and search for up to three providers at a time.
  • To get information about different areas, and compare to other areas, go to the Area Comparison page.

For more information about the Form 477 data collection or the data, please refer to the Related Links sidebar.

For Developers

Embed a Map Layer into your Website

This application includes map data for a total of 441 combinations of broadband technology and speed. All the map data are available as tilesets and available for public use; the Github link below will take you to a respository that contains all the code used to make use of the map data. It is not practical for the Commission to make all 441 technology/speed combinations available as web-map layers. However, we are able to do so for a limited number of layers. Instructions for accessing such layers is below.

Note: In order to access these layers, in the various formats below, you need a Mapbox access token. We are including a public access token in the links below since we want to make it easy for people to access broadband data in whatever format is most useful. However, if we find that the access token is being abused, and in particular being used to access maps unrelated to the broadband map, we will have no choice but to deactivate the token, and require end users to get their own token.
